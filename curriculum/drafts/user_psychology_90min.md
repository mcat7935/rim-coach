## A. 導入: なぜ心理を掘るのか (15分)

### Slide 26 — 章扉 + 問い (5分)
**Covers**: `K.MOT.Kakushin`, `K.QLY.PeerTest`

**タイトル**: ユーザー心理の捉え方 — 行動の列挙からの脱却
**サブ**: なぜ「ユーザーが買う・満足する」だけでは足りないのか

**問い (中央配置)**:
> あなたが描いた Value Cycle は、競合他社と区別がつきますか?

**左右比較ボックス**:
- 行動だけのループ → どの業界・どの企業でも似た構造になりがち
- 心理を埋めたループ → 自社ならではの差別化要素が浮かび上がる

**話し方メモ**: ここまでで簡易ループは描けるようになった。ただし「行動の連鎖」だけだと、楽天と Amazon が同じ図になってしまう。それを抜けるためのテーマが「心理」。

---

### Slide 27 — 「行動チェーン」だけのループの落とし穴 (5分)
**Covers**: `K.PSY.Drive`, `K.ANT.UX`

**タイトル**: 行動の連鎖だけでは、改善の打ち手が見えない

**左: 行動チェーン版**
```
商品を見る → 買う → 受け取る → 満足する → また買う
```
注釈: なぜ「また買う」のか? なぜ「満足する」のか? — 心理が見えないと、施策が打てない

**右: 心理を埋めた版**
```
「自分に合う商品が見つかる」期待 ↑
  → 検索行動
  → 「期待通り or 期待以上」達成感 ↑
  → 「次もここで探そう」信頼 ↑
  → 再利用
```
注釈: 同じ「再利用」でも、どの心理を高めるかで施策はまったく変わる

**まとめ**: 行動を駆動しているのは「心理状態の変化」。心理を見ないと、何を増やせばループが回るのか判断できない。

## B. 心理 = プロパティとして扱う (20分)

### Slide 29 — 心理状態 = 「変化するプロパティ」 (7分)
**Covers**: `K.NOT.Property`, `K.PSY.PropType.*` (4 種類全て)

**タイトル**: ユーザーの心理は「変化するプロパティ」として書ける

**定義**:
> プロパティ = アクター (ユーザー) が持つ、変化する状態。
> ゲームキャラクターの HP・スタミナ・経験値のようなもの。

**ユーザーの代表的な心理プロパティ**:
| 種類 | 例 |
|---|---|
| 期待・欲求系 | 期待値 / 欲求 / 関心 / 興味 |
| 不安・障壁系 | 不安感 / 抵抗感 / 面倒さ |
| 充足・確信系 | 達成感 / 満足度 / 信頼 / 安心感 |
| 想起・継続系 | 記憶の鮮明さ / 再利用意向 / 推奨意向 |

**復習 (既出 p.15 メルカリ例)**:
- 購入プロセスへの不安感 −
- 配送プロセスへの安心感 +
- 価格妥当性 +
- 返金可能性 +
- → 全体として「購入者への安心感」が形成される

これらは行動ではなく、すべて **心理プロパティ**。

---

### Slide 30 — 駆動プロパティ vs 結果プロパティ (7分)
**Covers**: `K.NOT.Property.Driver`, `K.NOT.Property.Result`, `K.PSY.Prop.Drv`, `K.PSY.Prop.Res`, `K.PSY.Prop.Pattern`

**タイトル**: プロパティには「駆動」と「結果」の 2 種類がある

**駆動プロパティ — 循環の起点**
- 微変化が次の行動を引き起こす
- 例: 欲求 / 期待値 / 関心 / 不安 (マイナス方向の駆動)

**結果プロパティ — 他の変化の結果として動く**
- 行動や体験の結果として変動する
- 例: 満足度 / 信頼 / 達成感 / 再利用意向

**重要な見分け方**:
- 駆動を見ないと「改善の入り口」が見えない
- 結果だけを追うと対症療法になる

**ループの基本パターン**:
```
[駆動プロパティ ↑] → [行動] → [結果プロパティ ↑] → [次サイクルの駆動プロパティ ↑] → …
```

---

### Slide 31 — 位置エネルギーの比喩 (6分)
**Covers**: `K.PSY.Potential`

**タイトル**: 「自然に次が起きる」位置を探す

**イメージ**:
> 水が高低差で自然に流れるように、心理状態が「高い位置」にあると、
> 次の行動が "意図せず" 起きる。

(既出 p.12 を心理プロパティの文脈で再利用)

**設計のための問いの型**:
> ユーザーの心理がどんな状態だったら、この行動が "自然に" 起きるか?

**例: 「商品を検索する」が自然に起きる心理状態は?**
- 「自分に合う商品があると期待している」
- 「探す手間が小さいと感じている」
- 「今すぐ必要だ、という関心が高い」

**逆に、位置エネルギーが低い心理**:
- 「期待していない」/「面倒くさい」/「不安がある」

**設計のコツ**:
> 行動を促す施策ではなく、行動が自然に起きる「位置エネルギーの高い心理」をつくる施策を考える。

## C. UX の 3 フェーズ整理 (20分)

### Slide 32 — 3 フェーズ整理: 利用前 / 利用中 / 利用後 (次回利用想起) (8分)
**Covers**: `K.PER.UX.Phases`, `K.PER.UX.Cycle`, `K.ANT.UX`

**タイトル**: UX の 3 フェーズ整理 — UX は一瞬ではない

**UX の全体は 3 つのフェーズに分かれる**:

| フェーズ | 起きていること | 代表心理プロパティ |
|---|---|---|
| **利用前** | 「使ってみたい」気持ちが立ち上がる | 期待 / 欲求 / 不安 |
| **利用中** | 期待が満たされる / 裏切られる | 達成感 / フラストレーション |
| **利用後 (次回利用想起)** | 「またあれ使おう」と思い出される | 信頼 / 記憶の鮮明さ / 想起確率 |

**よくあるアンチパターン**:
> 多くの Value Cycle は UI (利用中) だけを描いてしまう。
> その結果、「最初の一回」しか説明できないループになり、なぜ持続するのかが見えない。

**なぜ重要か**:
> 持続するサービスは、3 フェーズすべての心理プロパティが連鎖して循環している。
> 特に **利用後 (次回利用想起)** が弱いループは、リピートが説明できない。

---

### Slide 33 — 既知サービスを 3 フェーズで見直す (12分)
**Covers**: `K.PER.UX.Phases`, `K.PSY.PropType.Recall`

**タイトル**: 3 フェーズ × 既知サービス

**Amazon**
- 利用前: 「あれ欲しい → Amazon にありそう」想起 / 「最安で買えるはず」期待
- 利用中: 検索 → ワンクリック → 翌日着 → 「予想通り or 期待を超えた」達成感
- 利用後 (次回利用想起): 「Amazon で困ったことない」信頼蓄積 → 次回の想起確率↑

**YouTube**
- 利用前: 「あの動画の続きが見たい」/「暇つぶしに最適」想起
- 利用中: レコメンドで予想外の発見 → 「自分のためのフィード」感
- 利用後 (次回利用想起): 「YouTube 開けばなんかある」習慣化 → 再訪トリガー化

**Uber Eats**
- 利用前: 「今すぐ食べたい」+「外出したくない」
- 利用中: 注文 → 配達追跡 → 受け取り
- 利用後 (次回利用想起): 「面倒な時は Uber Eats」想起の強化

**共通点**:
> いずれも「利用後フェーズで想起されやすさを作っている」ことが、再利用ループの本質。
> 利用中の UI 改善だけでは、ここは育たない。

## D. 心理プロパティを発見する問い (15分)

### Slide 34 — 心理プロパティを掘り出す 4 つの問い (7分)
**Covers**: `K.PSY.FourQ`, `K.PSY.AntiBehavior`

**タイトル**: 自分のループに心理を埋めるための「4 つの問い」

| フェーズ | 問い |
|---|---|
| 利用前 | そのユーザーは何を期待していたか? どんな不安があったか? |
| 利用中 | 期待のうち、何が満たされたか? 何が裏切られたか? |
| 利用後 | 何が記憶として残ったか? どんな印象が積み上がったか? |
| 次のサイクルへ | 何が「次もまた」を引き起こすか? 何が「もう使わない」を引き起こすか? |

**使い方**:
自分が描いた Value Cycle の各ノードに、この 4 問のどれかが効くか確認する。
答えが「行動」しか出てこなければ、そのノードは心理に置き換えられないか考え直す。

---

### Slide 35 — 行動ループ → 心理駆動ループへの転換 (8分)
**Covers**: `K.PSY.Drive`, `K.PSY.Prop.Pattern`

**タイトル**: ループを "回す力" は心理プロパティの変化にある

**Before — 単純行動ループ**:
```
検索 → 購入 → 配達 → 受け取り → 再購入
```
(ループは閉じているが、なぜ回るのかが説明できない)

**After — 心理駆動ループ**:
```
期待値 ↑
  → 検索行動
  → 期待充足 → 達成感 ↑
  → 信頼 ↑
  → 想起されやすさ ↑
  → 次回の期待値 ↑ (ループ完成)
```

**良い Value Cycle の読み方の基準**:
> 各矢印が「A という心理が高まると、B という行動が自然に起き、その結果 C という心理が高まる」と読める。

> 行動はループの "動詞" だが、ループを回している "燃料" は心理プロパティの変化。

## E. ミニワーク (15分)

### Slide 36 — ミニワーク: 心理ジャーニーを描く
**Covers**: `K.PSY.FourQ`, `K.PSY.AntiBehavior` (実践適用)

**タイトル**: ミニワーク — 心理ジャーニーを描く (15分)

**お題**:
Amazon / YouTube / Uber Eats / メルカリ から 1 つ選ぶ

**手順**:
1. **個人 (5分)** — 選んだサービスについて、利用前 / 利用中 / 利用後 (次回利用想起) の各フェーズで、ユーザーの心理プロパティを 2-3 個ずつ書き出す
2. **ペア (5分)** — 互いに共有。「自分が見落としていたフェーズ」を発見する
3. **全体共有 (5分)** — 「利用後 (次回利用想起) で何が起きているか」を中心にいくつかピックアップして共有

**ルール**:
- 行動 (検索する、買う) ではなく、心理状態 (期待する、信頼する) で書く
- 1 つでもいいので「自分しか気づいてなさそうな心理」を入れてみる
