# RIM Variants

Multiple variants of RIM exist and are all considered valid. The methodology originator (Akitsugu Tsuchiya) has stated explicitly:

> "色んな RIM があって良いと思っているので"
> — Akitsugu Tsuchiya, Slack 2026-05-07

The judgment criterion is not "which type is correct" but whether the resulting diagram surfaces useful or debate-worthy points.

## Variant 1: Tsuchiya RIM — 山脈 + 峡谷 (mountain range + canyon)

### Approach
- Draw mountain peaks = **success points** (existing strengths) + their effects + the approaches that produced them
- To make these peaks into a connected mountain range (loop), find the **1 missing canyon (峡谷)** between them
- The canyon is the structural negative point that, if filled, would close a value loop

### Innovation type
**Asymptotic / incremental innovation (漸近的イノベーション).** The leap required to fill one specific canyon is bounded; the rest of the mountain range provides scaffolding.

### Trade-offs
- Starts from successful patterns → requires existing strengths to be identifiable
- Less suitable when no clear success points exist (greenfield, deep crisis, etc.)
- **AI intervention room: limited** — the canyon-bridging leap is human creativity

### Source
Akitsugu Tsuchiya, Slack dialogue 2026-05-07 with Akira Motomura, and interview 2026-05-16.

## Variant 2: Motomura inductive — 帰納的 negative-cycle

### Approach
- Start from stakeholder interview data (定性リサーチ)
- Apply grounded analysis: 切片化 (segmentation) → 構造化 (structuring) → 小テーマ → 中テーマ
- Surface the **As-Is negative cycle** (current negative loop) from the data: a closed loop where each negative theme reinforces the next
- Apply available technology / prototypes to **convert** the negative cycle into a positive value cycle

### Innovation type
**Often requires leap / breakthrough innovation (飛躍的イノベーション).** When the all-negative loop is dense, converting it to positive needs significant leverage. This is the "Othello endgame" risk (see `negative-loop-response.md`).

### Trade-offs
- Empirically rigorous (grounded in data)
- May surface "too many negatives chained" patterns that demand unrealistic single-flip interventions
- **AI intervention room: larger** — research processing, pattern detection, hypothesis generation can all be AI-supported

> "AI 介入の余地は大きそうだなと思った次第です。"
> — Akitsugu Tsuchiya about the Motomura variant, Slack 2026-05-07

### Aspiration toward procedural systematization
Motomura's stated direction:

> "リサーチのデータからの帰納的な練り上げ方の手続きと、そこから見える(負の)ループの立ち上げ方法を型にできれば、バリューサイクルの発見をより手続き的にできそう。"
> — Akira Motomura, Slack 2026-05-07

If the loop-extraction process can be systematized, value-cycle discovery becomes more procedurally tractable (and thus more AI-assistable at scale).

## Coexistence and complement

The two variants are **not in competition**; they often complement each other:

- Use **Motomura's inductive approach** to surface the As-Is negative cycle from real stakeholder data
- Then apply **Tsuchiya's mountain-range view** to identify which canyons to prioritize — converting the all-negative landscape into a peaks + targeted canyons diagram
- Apply **negative-loop response patterns** (mitigation / avoidance / substitution / acceptance — see `negative-loop-response.md`) to specific points

## Choosing a variant

Rough guidance (not prescriptive):

| Situation | Suggested variant |
|---|---|
| Existing success patterns are visible, looking for the next opportunity | Tsuchiya (mountain + canyon) |
| Data-rich situation with unclear or contradictory As-Is | Motomura (inductive) |
| AI assistance needed at scale, repeatable analysis desired | Motomura (more AI room) |
| Stakeholder buy-in needed via concrete data | Motomura (grounded in interviews) |
| Quick exploratory hypothesis generation by experienced practitioner | Tsuchiya (desktop approach) |
| Single canyon visible, leap is bounded | Tsuchiya |
| Large negative loop visible, need to find leverage points | Motomura + flip-search (see `negative-loop-response.md`) |

In practice, **diagrams may mix both variants** — and that is acceptable.

## Judgment criterion (across variants)

> "この図から有用（または関係者の多くが有用または議論価値がある）と思えるポイントを思いつくかどうか？が一番大事"
> — Akitsugu Tsuchiya, Slack 2026-05-07

What matters is whether the diagram surfaces useful (or debate-worthy) points for the next conversation. The variant type is secondary to this generative outcome.
