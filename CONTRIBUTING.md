# Contributing to rim-coach

Contributions welcome — prompt improvements, anonymized examples, anti-pattern reports.

## How to contribute

- **Prompt improvements**: open a PR with the change and a one-line rationale (why this nudges thinking better)
- **Examples**: clean, anonymized RIM diagrams (good or common-mistake) help others learn
- **Anti-patterns / misfires**: open an issue with the diagram (or anonymized version) and the prompt's output

## Constraints

- **No client-confidential material.** Use anonymized or fictional cases only.
- **Keep "hints not answers".** Prompts should nudge, not prescribe. Avoid wording that declares the right answer.
- **Cite sources.** When extracting principles from talks/articles, link the source.

## Background

Originator: **Akitsugu Tsuchiya** / Flying Penguins. The methodology has been refined over years of structural design practice (PS4 UX at Sony → Yumemi → Flying Penguins).
